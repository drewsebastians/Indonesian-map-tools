# Batch 2R Prompt 5 — Component contracts

## Public components now implemented

| Component | Contract |
| --- | --- |
| Public header | Brand, goal-oriented navigation, skip link, keyboard-operable mobile menu, and focus restoration after Escape. |
| Goal card | States whether a goal is available or upcoming. Only available cards are links. |
| Trust strip | Factual local-processing and source statements; it does not imply uploads, accounts, analytics, or current legal-boundary status. |
| Starting point | A local sample, guide, or file that helps a novice start without sending data anywhere. |
| CTA | One filled primary action; secondary actions are visibly quieter. |
| Public footer | Lightweight links to trust, policy, support, and region-data pages. |

## Contracts reserved for Prompt 6 and Prompt 7

The workflow header, form fields, dropzone, paste area, preview, match status, issue summary, region search, toolbar, inspector, data drawer, palette/legend controls, export panel, dialog, toast, confirmation, and state panels retain their Batch 2 behavior until the production workspace redesign. Prompt 6 must implement them using these rules:

- Name every icon-only control, keep a visible focus indicator, and announce meaningful status/errors.
- Keep error treatment actionable: explain the problem, identify the affected input/row, and offer a repair action without losing work.
- Keep the map and table equivalent: no region can be accessible only by pointer interaction.
- Use Option A's desktop rail; use Option C's map-first mobile sheets; use Option B's calm issue/error treatment.

The cross-workflow card for manual highlighting is a required Prompt 6 component: it must direct users with per-region values to the spreadsheet workflow and preserve the owner-approved meaning.
