# Batch 2R Prompt 7 — Owner manual test protocol

Status: **not yet run.** This protocol is for the owner or a facilitator; it does not claim a human usability result.

## Setup

- Recruit one non-GIS participant.
- Test once on a desktop browser and once on a narrow phone viewport/device.
- Provide a small CSV with a region name or official code and a value/category. Do not provide UI instructions beyond the task.
- Keep browser developer tools closed and do not use `?evidence=1` during a normal participant session.

## Tasks

1. Start from the home page and find **Highlight regions**. Highlight two regions, add source/period if useful, and export.
2. From the successful export, explain the suggested next workflow and dismiss it.
3. Start **Map spreadsheet data**. Explain where the data stays, paste/upload the sample, resolve an unmatched or ambiguous row, design the map, and export.
4. At export, find Save project, another export format, the relevant guide, and the one future-workflow suggestion. Explain why it is not available yet.
5. Trigger a safe recoverable error with an unsupported file or missing region column. Confirm the existing map/input remains present and find the next action.
6. On mobile, use the collapsed, medium, and expanded workspace sheet; open the data table; return to the map; and verify that no horizontal page scroll is needed.
7. With keyboard only on desktop, reach the input, matching controls, data table, export, and completion controls. Confirm visible focus and understandable status text.

## Facilitator notes

Record time to first valid preview, first resolved map, and first export. Record wording or controls the participant does not understand, whether the map remains prominent on mobile, and any recovery step that causes concern about data loss. Do not record spreadsheet values or personal information.

## Pass criteria

- The participant can explain the selected goal and take the next action without a tour.
- They can distinguish active work from an upcoming feature.
- They understand that the spreadsheet remains on the device.
- A recoverable error does not make them think the current map was deleted.
- Save/export stays more prominent than feature discovery.
