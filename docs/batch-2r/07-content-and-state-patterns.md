# Batch 2R Prompt 7 — Content and state patterns

## First use

The workspace starts with a short, inline explanation rather than a tour. It says that a map needs a region name or code and a value or category, identifies CSV, TSV, and XLSX as accepted inputs, offers a sample path, and states beside the input that the spreadsheet stays on the device. The same block links to the relevant guide and gives a brief keyboard hint. No account, upload service, analytics, or AI is involved.

## Empty and busy states

| State | Explanation | Primary next action |
| --- | --- | --- |
| No project / no autosave | No saved browser copy is available. | Start a map or open a project file. |
| No data | A spreadsheet has not been added yet. | Paste data, choose a file, or use the sample. |
| No selected region / no highlights | Nothing is selected for the manual map. | Search for a region or choose one on the map. |
| No unmatched rows | All current rows are resolved. | Apply the matches and continue. |
| No table results after a filter | The filter does not match a current row. | Clear or change the filter. |
| No export metadata | Source and period have not been added. | Add the available context, then export. |

The existing import, match, visualization, and export engines own their detailed empty text. The workspace shell adds a concise live status without changing their data or readiness guards. Local parsing, matching, and export use measurable status when one exists; otherwise they show a busy state with no invented percentage. A control is never disabled without an explanation of why.

## Recoverable errors

Every recovery message describes the affected scope, states whether current work is safe, and offers the nearest safe action. The existing domain engines remain authoritative for validation and do not overwrite the current map while an input is rejected.

| Category | Recovery direction |
| --- | --- |
| Unsupported, oversized, protected, corrupt, macro-bearing, or external-link XLSX | Keep the current map; use a plain supported XLSX or paste CSV/TSV. |
| Invalid worksheet or missing region column | Select/prepare a valid sheet with a region name or official code column. |
| Ambiguous, unmatched, or duplicate region | Review the row in Match regions; choose a candidate, correct the value, or ignore the row. |
| Invalid number | Correct the value column or use a category map. |
| Project migration warning or project replacement | Review the migration/update report or the existing replacement confirmation before continuing. |
| Local-storage quota or backup failure | The old browser copy remains untouched; save a project file and free browser storage before retrying. |
| Export failure or high-resolution memory risk | Keep the map open; use a smaller export or another available format. |
| Boundary asset failure | Do not apply a partial map; retry after the local boundary asset is available. |

The workspace recovery banner is intentionally additive: it repeats that existing work has not been cleared and returns focus to the relevant input. It does not hide a detailed error generated by the import or project engine.

## Completion

After a successful export, the workspace confirms the format, leaves **Save project** and **Export another format** available, links to the export guide, and offers exactly one contextual next workflow. The recommendation is not a modal, is dismissible, and never outranks saving or exporting. Reduced-motion users receive no completion animation.

## Accessibility and evidence mode

Status and recovery messages use live regions. The initial task and recovery action can be reached by keyboard, while the existing map/table link still permits selection without a pointer. The optional `?evidence=1` mode is disabled by default and records only local step names and timestamps for automated checks. It does not persist, transmit, or include spreadsheet values.

## Verification

`npm run test:batch2r:guidance` covers first-use copy, recovery focus and retained input, success recommendation limits and upcoming-workflow honesty, and the local-only evidence contract. Existing unit, smoke, trust, and workspace suites cover the detailed import, storage migration, boundary, export, keyboard, and responsive behavior.
